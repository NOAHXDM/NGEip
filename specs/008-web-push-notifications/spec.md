# 功能規格：瀏覽器推播通知

**功能分支**：`codex/dmit-2603-web-push`
**建立日期**：2026-07-12
**狀態**：已實作／複審中
**輸入**：Jira DMIT-2603「推播功能－競品分析與技術可行性評估」及產品決策收斂

## 文件範圍確認

- 本文件只定義使用者價值、產品邊界、使用者情境與驗收條件。
- 技術架構、Firebase Cloud Messaging、Token 生命週期、效能、風險與測試策略記錄於 `plan.md`。
- 本功能只服務一般性全體廣播，不包含會員分群、交易通知、個人化內容或送達保證。

## 背景與產品邊界

NGEip 使用者期望在未開啟網站時仍可收到一般性公告與提醒。由於 iPhone、iPad 與部分 Android 瀏覽器對 Web Push 有安裝與權限限制，系統需提供清楚的加入主畫面教學，並讓使用者自行決定此瀏覽器是否接收通知。

通知設定屬於「此瀏覽器／裝置」的本機偏好，不是帳號設定。管理者不得替使用者開啟通知，也不提供通知狀態檢視。系統不得宣稱通知必定送達。

## 使用者情境與測試

### 使用者故事 1－自行允許與停用通知（Priority: P1）

使用者可在通知設定頁自行允許、停用或重新啟用此瀏覽器的通知，不需接觸任何 Token 或工程設定。

**獨立驗證方式**：在支援通知的瀏覽器操作啟用與停用，確認原生權限視窗、頁面狀態與重新進入後的偏好一致。

**驗收情境**：

1. **Given** 此瀏覽器尚未決定通知權限，**When** 使用者點擊「啟用通知」並在原生視窗允許，**Then** 頁面顯示此裝置已啟用通知。
2. **Given** 此裝置已啟用通知，**When** 使用者點擊「停用此裝置的通知」，**Then** 系統停止此裝置訂閱，且重新開啟網站時不得自動恢復。
3. **Given** 瀏覽器已允許權限但使用者曾在站內停用，**When** 使用者再次點擊啟用，**Then** 系統重新建立訂閱且不要求操作 Token。

### 使用者故事 2－一次性授權引導（Priority: P1）

支援通知且尚未決定的使用者，在登入後看到一次產品說明，可選擇開啟或暫時不要；系統不得在每次登入重複打擾。

**獨立驗證方式**：清除網站資料後登入，分別測試開啟與暫不開啟，確認同一瀏覽器後續不重複顯示引導。

**驗收情境**：

1. **Given** 瀏覽器支援通知、權限為尚未決定且未看過引導，**When** 使用者登入，**Then** 顯示一次通知說明。
2. **Given** 使用者已關閉或完成引導，**When** 再次登入，**Then** 不重複顯示。
3. **Given** 瀏覽器權限已允許或封鎖，**When** 使用者登入，**Then** 不顯示原生權限引導。

### 使用者故事 3－加入主畫面教學與相容性指引（Priority: P2）

iPhone、iPad 與 Android 使用者可在通知設定頁查看簡短步驟，了解如何將網站加入主畫面並從主畫面開啟。

**獨立驗證方式**：以手機寬度檢視通知設定，確認兩平台教學清楚、可讀且不影響通知操作。

**驗收情境**：

1. **Given** 使用者查看通知設定，**When** 閱讀教學區塊，**Then** 可看到 iPhone／iPad 與 Android 各自的加入主畫面步驟。
2. **Given** 使用者裝置不支援通知，**When** 查看設定，**Then** 系統顯示相容性與 iOS 版本說明。
3. **Given** 瀏覽器已封鎖通知，**When** 查看設定，**Then** 系統指引使用者前往瀏覽器網站設定處理。

## 邊界案例

- 瀏覽器不支援 Web Push 時，不顯示無效的權限請求，只提供相容性說明。
- 清除網站資料會清除本機通知偏好；即使瀏覽器權限仍為允許，系統也不得自行重新訂閱。
- 登出不代表停用通知；此瀏覽器仍可接收非敏感的全體廣播。
- iOS／iPadOS 必須使用 16.4 以上版本，並從加入主畫面後的圖示啟動，才支援 Web Push。
- 網路、權限或訂閱建立失敗時，顯示一般失敗訊息，不向使用者暴露 Token、VAPID 或 Firebase 名詞。

## 功能需求

- **FR-001**：系統 MUST 由使用者主動操作後才要求瀏覽器通知權限。
- **FR-002**：系統 MUST 保存此瀏覽器的允許／停用意圖，且不得同步為帳號或管理者設定。
- **FR-003**：系統 MUST 將 Token 建立、更新與撤銷隱藏於內部生命週期，不提供 Token UI。
- **FR-004**：系統 MUST 在使用者明確停用後停止背景自動恢復訂閱。
- **FR-005**：登出 MUST NOT 修改此瀏覽器的通知偏好或訂閱。
- **FR-006**：管理者 MUST NOT 開啟或檢視使用者／裝置的通知狀態。
- **FR-007**：系統 MUST NOT 將 Token 或裝置通知狀態保存至使用者文件。
- **FR-008**：通知內容 MUST 限於非敏感、非個人化的全體廣播。
- **FR-009**：通知設定 MUST 提供 iPhone／iPad 與 Android 加入主畫面教學。
- **FR-010**：通知設定 Tab MUST 位於個人資料所有可見 Tab 的最後。

## 成功標準

- **SC-001**：支援裝置的使用者可在通知設定頁兩次操作內完成允許或停用。
- **SC-002**：使用者完成或關閉一次性引導後，同一瀏覽器後續登入不再自動顯示。
- **SC-003**：產品 UI、管理頁面及 Firestore 使用者文件均不顯示或保存 FCM Token。
- **SC-004**：通知生命週期、一次性提示及設定頁操作的自動化測試全部通過。
- **SC-005**：正式 Angular 建置成功，且 Service Worker 更新不受長期靜態快取阻擋。
